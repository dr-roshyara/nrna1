## Problem 1: Circular Redirect Between `/` and `/login`

### Symptoms

- Logs show alternating: `GET /` → `GET /login` → `GET /` → `GET /login` (repeat)
- User cannot complete login process
- Session gets stuck in redirect loop

### Root Cause

The 'guest' middleware on login route (line 109 in `routes/web.php`) is causing conflicts:

```php
Route::middleware('guest')->group(function () {
    Route::get('/login', [\App\Http\Controllers\Auth\LoginController::class, 'show'])->name('login');
});
```

**What happens:**
1. User navigates to `/login`
2. SetLocale middleware runs (global)
3. TenantContext middleware runs for authenticated users
4. Guest middleware checks: Are you logged in?
   - If YES → Redirect to intended location (or `/`)
   - If NO → Proceed to login form
5. But `/` (dashboard) might check auth and redirect back to `/login`
6. **Loop created**

### Fix 1: Add Safe Redirect for Authenticated Users on Login

**File:** `app/Http/Controllers/Auth/LoginController.php`

Add this check to the `show()` method:

```php
public function show()
{
    // If already authenticated, don't show login form
    // Redirect to intended location or dashboard
    if (auth()->check()) {
        return redirect()->intended(route('electiondashboard'));
    }

    return Inertia::render('Auth/Login', [
        'canResetPassword' => \Route::has('password.request'),
        'canRegister' => \Route::has('register'),
        'status' => session('status'),
    ]);
}
```

### Fix 2: Update Guest Middleware Configuration

**File:** `routes/web.php`

Remove the guest middleware wrapper and apply it more carefully:

```php
// ✅ CORRECT: Apply guest middleware only where needed
Route::get('/login', [\App\Http\Controllers\Auth\LoginController::class, 'show'])->name('login')->middleware('guest');
Route::post('/login', [\App\Http\Controllers\Auth\LoginController::class, 'store'])->middleware('guest');
```

## Problem 2: Login Page Missing Language Selector

### Symptoms

- `/login` page shows NO language selection
- Users default to `de` (German)
- No way to change language before logging in
- Frontend can't set locale cookie until after login

### Root Cause

The Login component doesn't include a language selector. The `SetLocale` middleware determines language from:
1. Locale cookie (user preference)
2. Session locale
3. Config default

But there's no UI to SET the locale before login.

### Fix: Add Language Selector to Login Component

**File:** `resources/js/Pages/Auth/Login.vue`

Add language selector at the top of the login form:

```vue
<template>
  <div class="auth-container">
    <!-- ✅ NEW: Language Selector -->
    <div class="language-selector">
      <label>Language / Sprache / भाषा:</label>
      <select @change="changeLanguage" :value="currentLocale" class="language-select">
        <option value="en">English</option>
        <option value="de">Deutsch</option>
        <option value="np">नेपाली</option>
      </select>
    </div>

    <!-- Login Form -->
    <form @submit.prevent="handleLogin">
      <h1>{{ $t('pages.auth.login.title') }}</h1>

      <div class="form-group">
        <label for="email">{{ $t('pages.auth.login.email') }}</label>
        <input
          id="email"
          v-model="form.email"
          type="email"
          :placeholder="$t('pages.auth.login.email_placeholder')"
          required
        />
      </div>

      <!-- ... rest of form ... -->
    </form>
  </div>
</template>

<script>
export default {
  name: 'LoginComponent',
  data() {
    return {
      form: {
        email: '',
        password: '',
      },
    };
  },
  computed: {
    currentLocale() {
      return this.$i18n.locale;
    },
  },
  methods: {
    changeLanguage(event) {
      const locale = event.target.value;

      // 1. Update Vue i18n locale
      this.$i18n.locale = locale;

      // 2. Set locale cookie (persists for 1 year)
      document.cookie = `locale=${locale}; path=/; max-age=31536000; SameSite=Lax`;

      // 3. Store in session via hidden API call (optional, for server-side backup)
      fetch('/api/set-locale', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').content,
        },
        body: JSON.stringify({ locale }),
      }).catch(err => console.log('Session locale update skipped:', err));
    },

    async handleLogin() {
      // Login logic...
    }
  }
};
</script>

<style scoped>
.language-selector {
  margin-bottom: 20px;
  display: flex;
  gap: 10px;
  align-items: center;
}

.language-select {
  padding: 8px 12px;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 14px;
  cursor: pointer;
}

.language-select:hover {
  border-color: #999;
}
</style>
```

### Fix: Create Optional API Endpoint for Session Locale

**File:** `app/Http/Controllers/LocaleController.php` (new)

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class LocaleController extends Controller
{
    /**
     * Set user's locale preference in session
     *
     * This is a backup to the cookie-based locale selection.
     * Frontend sends preferred locale, we store it in session.
     */
    public function setLocale(Request $request)
    {
        $locale = $request->validate(['locale' => 'required|in:de,en,np'])['locale'];

        // Store in session
        $request->session()->put('locale', $locale);

        return response()->json([
            'success' => true,
            'locale' => $locale,
            'message' => 'Locale set successfully',
        ]);
    }
}
```

**File:** `routes/api.php` (add)

```php
Route::post('/set-locale', [\App\Http\Controllers\LocaleController::class, 'setLocale']);
```
