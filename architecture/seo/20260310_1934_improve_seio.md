## ✅ **SEO IMPROVEMENTS FOR LARAVEL + INERTIA + VUE3 PROJECT**

Great question! SEO with Inertia + Vue3 requires a **hybrid approach** since content is client-rendered. Here are actionable improvements:

---

## 📋 **CRITICAL: SERVER-SIDE RENDERING (SSR)**

### **1. Enable SSR for Inertia**
```bash
npm install @inertiajs/server @vue/server-renderer
```

```js
// vite.config.js
import inertia from '@inertiajs/inertia-vue3'
import ssr from '@inertiajs/server'

export default {
    plugins: [
        inertia({
            ssr: true  // ✅ Enable SSR
        })
    ]
}
```

### **2. Create SSR Entry Point**
```js
// resources/js/ssr.js
import { createSSRApp, h } from 'vue'
import { renderToString } from '@vue/server-renderer'
import { createInertiaApp } from '@inertiajs/inertia-vue3'
import createServer from '@inertiajs/server'

createServer((page) => createInertiaApp({
    page,
    render: renderToString,
    resolve: name => require(`./Pages/${name}`),
    setup({ app, props, plugin }) {
        return createSSRApp({
            render: () => h(app, props)
        }).use(plugin)
    }
}))
```

### **4. Create SEO Service Provider**
```php
// app/Providers/SEOServiceProvider.php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Inertia\Inertia;

class SEOServiceProvider extends ServiceProvider
{
    public function boot()
    {
        Inertia::share('seo', function () {
            $route = request()->route()->getName();
            
            $meta = [
                'home' => [
                    'title' => 'Online Voting Platform | Secure Digital Elections',
                    'description' => 'Secure online voting platform for organizations, associations, and elections. GDPR compliant, end-to-end encrypted.',
                    'keywords' => 'online voting, digital elections, secure voting platform',
                    'og_image' => asset('images/og-home.jpg'),
                ],
                'election.show' => [
                    'title' => fn($election) => "Vote in {$election->name} | Secure Online Election",
                    'description' => fn($election) => "Cast your vote securely in {$election->name}. End-to-end encrypted online voting.",
                ],
                'vote.create' => [
                    'title' => 'Cast Your Vote | Secure Online Voting',
                    'description' => 'Select your candidates and cast your vote securely. Your vote is anonymous and encrypted.',
                    'robots' => 'noindex,nofollow', // Voting pages shouldn't be indexed
                ],
            ];

            return $this->generateMeta($meta[$route] ?? []);
        });
    }
}
```

### **5. Meta Component**
```vue
<!-- resources/js/Components/SEO/MetaTags.vue -->
<template>
    <Head>
        <title>{{ meta.title }}</title>
        <meta name="description" :content="meta.description" />
        <meta name="keywords" :content="meta.keywords" />
        
        <!-- Open Graph -->
        <meta property="og:title" :content="meta.title" />
        <meta property="og:description" :content="meta.description" />
        <meta property="og:image" :content="meta.og_image" />
        <meta property="og:url" :content="meta.url" />
        <meta property="og:type" content="website" />
        
        <!-- Twitter -->
        <meta name="twitter:card" content="summary_large_image" />
        <meta name="twitter:title" :content="meta.title" />
        <meta name="twitter:description" :content="meta.description" />
        <meta name="twitter:image" :content="meta.og_image" />
        
        <!-- Robots -->
        <meta name="robots" :content="meta.robots || 'index,follow'" />
        
        <!-- Canonical -->
        <link rel="canonical" :href="meta.canonical || meta.url" />
    </Head>
</template>

<script>
import { Head } from '@inertiajs/vue3'
import { computed } from 'vue'

export default {
    components: { Head },
    props: {
        custom: Object
    },
    setup(props) {
        const meta = computed(() => ({
            ...usePage().props.seo,
            ...props.custom
        }))
        
        return { meta }
    }
}
</script>
```

## 📋 **STRUCTURED DATA / SCHEMA.ORG**

### **6. Dynamic Schema Generator**
```php
// app/Services/SchemaService.php

namespace App\Services;

class SchemaService
{
    public function generateElectionSchema($election)
    {
        return [
            '@context' => 'https://schema.org',
            '@type' => 'Event',
            'name' => $election->name,
            'description' => $election->description,
            'startDate' => $election->start_date->toIso8601String(),
            'endDate' => $election->end_date->toIso8601String(),
            'eventStatus' => 'https://schema.org/EventScheduled',
            'eventAttendanceMode' => 'https://schema.org/OnlineEventAttendanceMode',
            'location' => [
                '@type' => 'VirtualLocation',
                'url' => config('app.url')
            ],
            'organizer' => [
                '@type' => 'Organization',
                'name' => $election->organisation->name,
                'url' => route('organisation.show', $election->organisation)
            ]
        ];
    }

    public function generateOrganizationSchema()
    {
        return [
            '@context' => 'https://schema.org',
            '@type' => 'Organization',
            'name' => config('app.name'),
            'description' => 'Secure online voting platform for organizations',
            'url' => config('app.url'),
            'logo' => asset('images/logo.png'),
            'sameAs' => [
                'https://twitter.com/yourhandle',
                'https://linkedin.com/company/yourcompany',
            ],
            'contactPoint' => [
                '@type' => 'ContactPoint',
                'telephone' => config('app.phone'),
                'contactType' => 'customer service',
                'availableLanguage' => ['English', 'German']
            ]
        ];
    }
}
```

### **7. Add Schema to Pages**
```vue
<!-- In AppLayout.vue or specific pages -->
<script>
import { usePage } from '@inertiajs/vue3'

export default {
    setup() {
        const page = usePage()
        
        const schema = computed(() => [
            page.props.organizationSchema,
            page.props.electionSchema
        ].filter(Boolean))
        
        return { schema }
    }
}
</script>

<template>
    <div>
        <MetaTags />
        
        <!-- Structured Data -->
        <script v-for="data in schema" :key="data['@type']" type="application/ld+json">
            {{ JSON.stringify(data) }}
        </script>
        
        <slot />
    </div>
</template>
```

## 📋 **SITEMAP & ROBOTS**

### **8. Dynamic Sitemap Generator**
```php
// routes/web.php
Route::get('/sitemap.xml', function () {
    $elections = Election::where('status', 'active')
        ->where('type', 'real')
        ->get();
    
    $posts = Post::with('election')->get();
    
    return response()->view('sitemap', [
        'elections' => $elections,
        'posts' => $posts,
        'baseUrl' => config('app.url')
    ])->header('Content-Type', 'text/xml');
});
```

```blade
<!-- resources/views/sitemap.blade.php -->
<?= '<?xml version="1.0" encoding="UTF-8"?>' ?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
    <!-- Static pages -->
    <url>
        <loc>{{ $baseUrl }}</loc>
        <changefreq>daily</changefreq>
        <priority>1.0</priority>
    </url>
    
    <!-- Election pages -->
    @foreach($elections as $election)
    <url>
        <loc>{{ route('election.show', $election) }}</loc>
        <lastmod>{{ $election->updated_at->toAtomString() }}</lastmod>
        <changefreq>weekly</changefreq>
        <priority>0.8</priority>
    </url>
    @endforeach
    
    <!-- Post/Information pages -->
    @foreach($posts as $post)
    <url>
        <loc>{{ route('post.show', $post) }}</loc>
        <lastmod>{{ $post->updated_at->toAtomString() }}</lastmod>
        <changefreq>weekly</changefreq>
        <priority>0.6</priority>
    </url>
    @endforeach
</urlset>
```

## 📋 **PERFORMANCE SEO**

### **10. Preload Critical Resources**
```php
// app/Http/Middleware/PreloadAssets.php

namespace App\Http\Middleware;

use Closure;

class PreloadAssets
{
    public function handle($request, Closure $next)
    {
        $response = $next($request);
        
        $response->header('Link', [
            '<'.asset('css/app.css').'>; rel=preload; as=style',
            '<'.asset('js/app.js').'>; rel=preload; as=script',
            '<'.asset('fonts/inter.woff2').'>; rel=preload; as=font; type=font/woff2; crossorigin'
        ]);
        
        return $response;
    }
}
```

## 📋 **KEYWORD & CONTENT OPTIMIZATION**

### **12. Keyword-Focused Routes**
```php
// Instead of:
Route::get('/election/{id}', [ElectionController::class, 'show'])

// Use SEO-friendly slugs:
Route::get('/election/{election:slug}', [ElectionController::class, 'show'])

// In Election model:
public function getRouteKeyName()
{
    return 'slug';
}
```

### **13. Generate SEO-Friendly Slugs**
```php
// app/Models/Election.php
use Spatie\Sluggable\HasSlug;
use Spatie\Sluggable\SlugOptions;

class Election extends Model
{
    use HasSlug;
    
    public function getSlugOptions(): SlugOptions
    {
        return SlugOptions::create()
            ->generateSlugsFrom(['name', 'organisation.name'])
            ->saveSlugsTo('slug')
            ->slugsShouldBeNoLongerThan(50)
            ->usingSeparator('-');
    }
}
```
