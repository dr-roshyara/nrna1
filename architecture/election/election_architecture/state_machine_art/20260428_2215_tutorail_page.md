# Claude CLI Prompt: Public Election Journey Tutorial Page

---

## Role: Senior Developer — Election Platform

You are building a **public-facing tutorial page** at `/election-journey` that explains the complete election lifecycle. This page is independent of any specific election — it's a guide for anyone to understand how elections work on the platform.

---

## Context

- The platform has a 7-state election lifecycle: `draft → pending_approval → administration → nomination → voting → results_pending → results`
- The management page at `/elections/{slug}/management` shows a state machine panel with badges like "Nomination Locked" that users find confusing
- This tutorial page explains what each phase means, who performs each action, and what status badges/locks mean
- The page is **public** — no authentication required
- There should be a **navigation link** from the Management page to this guide

## Step 2: Create the Tutorial Page

File: `resources/js/Pages/Public/ElectionJourney.vue`

### Design Requirements:
- **Public layout** (PublicDigit header/footer, no auth required)
- **Visual timeline** showing all 7 states in order
- **Each phase section** includes:
  - Phase name with icon
  - Description of what happens
  - Who performs actions (Owner, Chief, Platform Admin, System)
  - What "locked" means for that phase (if applicable)
  - Business rules (e.g., ≤40 voters = free, >40 = paid)
- **Glossary section** explaining common terms:
  - "Phase Locked" / "Nomination Locked" — means dates can't be edited, not that the phase is stuck
  - "Pending Approval" — waiting for platform admin review
  - "Expected Voters" — declared capacity that determines free vs paid
- **Subscription info section**:
  - Free: ≤ 40 expected voters → auto-approved
  - Paid: > 40 expected voters → platform admin approval required
- **Responsive design** — mobile-friendly, single column on small screens

### Content for Each Phase:

| Phase | Icon | Description | Who Acts | Locked Means |
|-------|------|-------------|----------|--------------|
| Draft | 📝 | Election is being created. Set name, expected voters, phase dates. | Organisation Owner/Admin | N/A — editable |
| Pending Approval | ⏳ | Large elections (>40 voters) wait for platform admin review. Small elections skip this. | Platform Admin | Cannot edit — waiting for decision |
| Administration | ⚙️ | Create posts/positions, import voters, manage election committee. | Chief, Deputy | "Admin Locked" — phase completed, no more edits |
| Nomination | 📋 | Candidates submit applications. Chief reviews and approves/rejects. | Candidates, Chief | "Nomination Locked" — candidates fixed, dates frozen. **Open Voting still available!** |
| Voting | 🗳️ | Voters cast their votes. Voting period is active. | Voters | "Voting Closed" / "In Progress" — dates locked, voting underway |
| Results Pending | 📊 | Voting ended. Results being counted and verified. | System / Chief | "Results Pending" — waiting for publication |
| Results | ✅ | Results published. Public can view final outcome. | Everyone | "Results Published" — final, immutable |
