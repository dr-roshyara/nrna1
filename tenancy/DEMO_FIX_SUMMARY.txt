PROBLEM IDENTIFIED
═════════════════════════════════════════════════════════════════════════════

Before Creating Organisation:
  ✅ User has organisation_id = NULL
  ✅ Can access /election/demo/start
  ✅ Demo election queries return results

After Creating Organisation:
  ❌ User now has organisation_id = 1, 2, 3, etc.
  ❌ Tries to access /election/demo/start
  ❌ Demo election queries return NOTHING
  ❌ Redirected to dashboard with error message

═════════════════════════════════════════════════════════════════════════════

ROOT CAUSE
═════════════════════════════════════════════════════════════════════════════

Election model uses BelongsToTenant trait which adds a global scope:

  session('current_organisation_id') === null
    → Query: WHERE organisation_id IS NULL

  session('current_organisation_id') === 1
    → Query: WHERE organisation_id = 1

Problem:
  • Demo election created with organisation_id = NULL
  • When user creates organisation, session changes to their org_id
  • Global scope now hides demo election (NULL ≠ user's org_id)
  • Demo election becomes invisible to users with organisations

═════════════════════════════════════════════════════════════════════════════

THE FIX
═════════════════════════════════════════════════════════════════════════════

Demo elections are UNIVERSAL testing resources and should be accessible
to ALL users regardless of organisation context.

Solution: Use withoutGlobalScopes() when querying for demo elections

  OLD (BROKEN):
    $demoElection = Election::where('type', 'demo')->first();

  NEW (FIXED):
    $demoElection = Election::withoutGlobalScopes()
        ->where('type', 'demo')
        ->first();
